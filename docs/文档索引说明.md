## 文档维护规则

### 部署任务文档 (deployment_progress)
1. 每个任务文档必须包含：任务概述、模块职责、详细部署说明、依赖关系、测试验证、验收标准
2. 任务完成后更新状态
3. 任务之间不能有功能重叠
4. 避免隐式依赖

### 项目说明文档 (project_docs)
1. 每个说明文档必须包含：概述、详细说明、示例代码
2. 文档内容与实际代码保持一致
3. 定期更新以反映项目变化

## 文档贡献指南

### 创建新部署任务文档
1. 在 deployment_progress 文件夹创建新文档
2. 使用 "序号-任务名称.md" 或 "序号-英文任务名称.md" 命名格式
3. 遵循任务文档模板结构
4. 更新本文档索引

### 更新现有文档
1. 修改对应文档内容
2. 更新文档版本和日期
3. 同步更新相关文档
4. 通知相关人员

## 文档质量标准

### 内容要求
- 清晰的模块职责划分
- 详细的部署步骤和代码示例
- 完整的测试验证方法
- 明确的验收标准

### 格式要求
- 使用Markdown格式
- 中文标题和内容
- 代码块有语法高亮
- 表格格式规范

### 维护要求
- 定期检查文档有效性
- 及时更新以反映项目变化
- 清理过时或冗余文档
- 保持文档与代码同步
