## 规则内容概要

### 项目根目录强制认定规则
- 项目唯一根目录: `my_ai_popup_project/`
- 所有路径解析、文件扫描均以该目录为起点

### 项目全局联动与一致性规则
- 功能修改时必须同步分析关联模块
- 任何修改不得是"孤立修改"
- 重复逻辑必须统一优化

### 原文件优先优化原则
- 优先在原有文件/原有脚本中进行修改
- 禁止无必要地新建重复文件
- 禁止逻辑分叉、功能分裂

### 子项目与目录逐级识别规则
- 必须从项目根目录开始逐级识别
- 识别深度要求: 文件夹 → 子文件夹 → 文件 → 代码 → 最后一行

### 终端与执行环境识别规则
- 每次执行终端命令前必须明确终端类型
- 禁止在错误终端执行命令

### 错误自动修正与持续优化规则
- 发现错误时必须自动分析原因并修复
- 同步修复所有受影响模块

### 执行前分析与用户确认机制
- 任何实际部署/执行/修改代码前必须分析+复述+用户确认

### 上下文完整识别规则
- 处理脚本/配置/文档/代码文件时必须完整理解上下文
- 禁止断章取义
